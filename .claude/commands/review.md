# コードレビュー

`git diff HEAD` の差分を取得し、以下の観点でレビューしてください。

## チェック項目
- [ ] Clean Architecture の層境界を侵犯していないか
- [ ] BLE処理に適切なエラーハンドリングがあるか
- [ ] 非同期処理の競合状態（race condition）が発生しないか
- [ ] Riverpod の状態更新が正しいライフサイクルで行われているか
- [ ] テストが書かれているか（Domain層は必須）
- [ ] `dart analyze` のエラーがないか
- [ ] コミットメッセージが Conventional Commits 規約に従っているか

問題点は重要度（高/中/低）を付けて列挙し、修正コードの提案も行うこと。
